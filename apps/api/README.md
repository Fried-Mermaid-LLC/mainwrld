# apps/api тАФ NestJS backend (Cloud Run)

> Part of the **mainwrld** monorepo. For the architecture overview, data model, end-to-end
> flows, local development, and CI/CD see the [root README](../../README.md). This file is the
> in-depth reference for this workspace.

REST API backing `apps/app`, fronted by Firebase Admin. NestJS 11 on Node 24, deployed to Cloud Run from [Dockerfile](Dockerfile). Routes under `/api/v1/...`; health probes at root. Every write the client used to do directly (and every legacy callable) was ported here so the **server owns auth, moderation, points, and monetization**.

### Bootstrap & global infra
- [src/main.ts](src/main.ts) тАФ `NestExpressApplication` with `rawBody: true` (Stripe HMAC). Global prefix `api` (excludes `healthz`/`readyz`), URI versioning (`defaultVersion:'1'` тЖТ `/api/v1`). Global `ValidationPipe` is `whitelist + transform` but **NOT** `forbidNonWhitelisted` (extra fields dropped silently; server-managed fields protected by per-domain denylists instead). Body limit 5 MB. CORS from `CORS_ORIGINS` (empty тЗТ reflect). Swagger at `/api/docs` only when `NODE_ENV !== production`. `nestjs-pino` logging.
- [src/app.module.ts](src/app.module.ts) тАФ imports every feature module + `LoggerModule`. Three app providers, **order matters**: `AuthGuard` тЖТ `RolesGuard` тЖТ `AllExceptionsFilter`.
- [src/infra/config/configuration.ts](src/infra/config/configuration.ts) тАФ zod-validated flat env тЖТ typed nested config (`config.get('firebase',{infer:true})`). Secrets optional (boots in local dev before each domain is wired). Loaded globally by [config.module.ts](src/infra/config/config.module.ts).
- [src/infra/firebase/firebase.module.ts](src/infra/firebase/firebase.module.ts) тАФ `@Global`. Provides DI tokens (`FIREBASE_APP`, `FIRESTORE`, `FIREBASE_AUTH`, `FIREBASE_STORAGE`, `FIREBASE_MESSAGING`, `FIREBASE_DB` from [firebase.constants.ts](src/infra/firebase/firebase.constants.ts)). Credentials: a service-account JSON file **only if `FIREBASE_SERVICE_ACCOUNT_PATH` is set and exists** (local dev), else `applicationDefault()` (Cloud Run runtime SA). Firestore `ignoreUndefinedProperties:true`. `firebase.constants.ts` also exports `COLLECTIONS` тАФ the single source of truth for collection names.
- [src/common/filters/all-exceptions.filter.ts](src/common/filters/all-exceptions.filter.ts) тАФ normalizes errors to `{statusCode, code, message}`. `STATUS_TO_CODE` maps HTTP status тЖТ stable string codes (legacy `HttpsError` vocabulary the client branches on); services can throw a Nest exception with a `{code, message}` payload to carry a domain code (e.g. `payouts-required`).

### Auth, roles & SSE (cross-cutting)
- [auth.guard.ts](src/infra/auth/auth.guard.ts) тАФ **global** `AuthGuard`. `verifyIdToken(token, true)` (`checkRevoked` closes the window after a ban revokes refresh tokens). Populates `req.user: AuthUser` (`uid, email?, username?, admin, banned`). `@Public()` routes skip it.
- [roles.guard.ts](src/infra/auth/roles.guard.ts) тАФ runs after AuthGuard; `@Roles('admin')` checks the `admin` claim. Decorators in [auth.decorators.ts](src/infra/auth/auth.decorators.ts) (`@Public()`, `@Roles(...)`, `@CurrentUser()`). [require-username.ts](src/infra/auth/require-username.ts) throws `failed-precondition` when the `username` claim is missing.
- **SSE** ([stream.service.ts](src/modules/stream/stream.service.ts)): per-connection Firestore admin `onSnapshot` (no Pub/Sub тАФ Firestore is the cross-instance source on Cloud Run). Filters docs predating `connectedAt`; `ping` every 25s. Authed by the global AuthGuard via the Bearer the client sends through `fetch-event-source`.

### Stripe / payments (cross-cutting)
- [stripe.service.ts](src/modules/payments/stripe.service.ts) тАФ `forMode(mode)` picks `stripeSecretKey` (`'live'`) vs `stripeTestSecretKey`; client sends `mode:'live'` only in prod builds.
- **Connect** ([payments.service.ts](src/modules/payments/payments.service.ts)): Express accounts (`transfers` capability), account/login links, balance, status sync onto `users/{uid}`. `isAccountGone()` detects a stored `acct_тАж` from the other key and re-onboards instead of 500ing. `safeOrigin()` forces HTTPS.
- **Checkout** (`createBookCheckout`): 70/30 split via `transfer_data.destination` + `transfer_data.amount` (fixed transfer, **not** `application_fee_amount`, so collected sales tax stays with the platform = merchant of record). Stripe Tax (`txcd_10302000`, `tax_behavior:'exclusive'`). In-app coupon тЖТ one-time Stripe coupon.
- **Webhook** ([stripe-webhook.service.ts](src/modules/webhooks/stripe-webhook.service.ts)): `constructEvent` tries live then test secret. **Idempotency** = no-op if `stripeEvents/{event.id}` exists. Branches on `metadata.kind === 'book_purchase'` (grants `purchasedBookIds`/`ownedBookIds`, writes `bookPurchases/{paymentIntentId}`) vs `metadata.sku` (points / `premium_yearly` / coupon тАФ **SKU maps kept in sync with [iap.service.ts](src/modules/iap/iap.service.ts) and `app/iap.ts`**). Returns 500 on failure so Stripe retries.
